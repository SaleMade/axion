PAYT — Plataforma de vendas
Integração nativa para automatizar pagamento, status do pedido e funil.
1. Configuração no PAYT

Cole esta URL no painel PAYT > Postbacks. Marque todos os 13 eventos pra aproveitar 100% das automações.

https://axion-api.axion-dash.workers.dev/webhook/payt/b562d560380649cbc6c8ade3550eb7f8
✓ Backend pronto — recebe postbacks da PAYT, identifica leads por CPF/WhatsApp/Email e aplica as automações abaixo.
Chave única: b562d560380649cbc6c8ade3550eb7f8
2. Mapeamento de eventos PAYT → Etapa do CRM

Configure para qual etapa do CRM cada evento da PAYT deve mover o lead. Salvo automaticamente.

3. Simular evento PAYT (teste local)

Roda só na sua Dash, sem mandar pro servidor. Útil pra testar o mapeamento de etapas.

4. Últimos eventos recebidos do servidor

Quando a PAYT (ou qualquer integração) bate na URL acima, registramos aqui. Útil pra confirmar que o teste chegou.

📦
Webhook do Fornecedor
Receba leads automaticamente quando seu fornecedor / plataforma de captação criar um novo registro.
URL para o fornecedor

Envie esta URL pra quem precisa cadastrar leads na sua Dash. Cada chamada gera 1 lead automático em "A Enviar" com tag fornecedor.

https://axion-api.axion-dash.workers.dev/webhook/fornecedor/frn_a47c9f8e3b21d5046ec8fa9d2b7e4513
Eventos suportados (criação + status)

O fornecedor pode enviar tanto criação de pedido quanto atualizações de status. Cada evento move o lead automaticamente conforme o mapeamento abaixo.

new_orderPedido criado → cria lead em A Enviar
status: pendingAguardando pagamento → spg Pendente
status: confirmedPago / confirmado → spg Pago ou Aguardando confirmação (se COD)
tracking_createdRastreio gerado → atualiza campo Cód. rastreio
shippedEnviado → coluna Enviado
out_for_deliverySaiu pra entrega → tag entrega-hoje
deliveredEntregue → coluna Retirada / Pago
canceled / refunded / chargebackDevoluções → coluna Cancelado
Como o fornecedor envia (POST JSON)

Aceita 2 formatos: achatado (raiz) ou aninhado (estilo PAYT, com order e address). Use o que for mais conveniente pro fornecedor.

// Formato aninhado (PAYT-like) — recomendado:
{
  "event": "new_order",                 // ou status_update, shipped, delivered, etc
  "order": {
    "id": "ABC-123",                    // external_id (evita duplicar)
    "status": "confirmed",              // pending|confirmed|paid|shipped|delivered|canceled
    "client_name": "João Silva",        // obrigatório p/ criar
    "client_whatsapp": "11999998888",
    "cpf": "12345678901",
    "client_email": "joao@email.com",
    "total_amount": 397.00,
    "payment_modality": "on_delivery",  // "on_delivery"=COD, "prepaid"=antecipado
    "payment_method": "pix",
    "tracking_code": "BR123456789",
    "brand": "Produto X",
    "address": {
      "cep": "01310100", "street": "Av. Paulista", "number": "1000",
      "complement": "ap 5", "neighborhood": "Bela Vista",
      "city": "São Paulo", "state": "SP"
    }
  }
}

// Formato achatado (alternativa):
{
  "event": "new_order", "external_id": "ABC-123",
  "nome": "João Silva", "cpf": "12345678901",
  "telefone": "11999998888", "valor": 397.00,
  "modalidade": "antecipado", "produto": "Produto X"
}
Resposta da API
{
  "ok": true,
  "event_raw": "new_order",
  "event_mapped": "aguardando_confirmacao",
  "status": "confirmed",
  "action": "created",  // ou "updated" / "skipped"
  "lead_id": 42
}
Mapeamento dos eventos (Fornecedor)

Configure o que cada evento faz no CRM. Por padrão usa o mesmo mapeamento da PAYT — você pode customizar separado aqui.

💡 Dica de segurança: a chave da URL é como uma senha — só compartilhe com o fornecedor de confiança. Se vazar, você pode trocar editando FORN_TOKEN_DEFAULT em backend/worker.js e fazendo deploy de novo.
